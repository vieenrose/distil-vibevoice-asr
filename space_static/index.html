🎙️ zh-TW / English meeting transcription + speaker diarization

Luigi/moss-transcribe-diarize-zhtw (0.9B, fine-tuned from MOSS-Transcribe-Diarize) transcribes hours-long Traditional-Chinese / English meetings in 5-minute windows, then ECAPA-TDNN speaker embeddings + global clustering keep one consistent set of speaker labels across the whole meeting. Everything below was produced by that pipeline on real Taiwan Legislative Yuan recordings — pick a clip, press play, click any line to jump.

How it works

Audio is split into 5-minute windows; each window is transcribed in one pass by the 0.9B model (Whisper-Medium encoder + Qwen3-0.6B decoder), producing [start][Sxx]text[end] segments whose speaker labels are only consistent within a window. Every segment is then embedded with ECAPA-TDNN, and one global agglomerative-clustering pass (segments ≥ 3 s, cosine 0.45, average linkage; shorter segments snap to the nearest cluster centroid) links speakers across windows.

Chunking is not just a memory trick: a single pass over the 30-minute meeting truncates (65 vs 145 segments, DER 0.556 vs 0.474) — windows + global linking beat one-shot decoding even on a workstation GPU. The same design runs on phones (~2 GB int8) via a sherpa-onnx C++ port.
